The Big Picture

Your product search engine is live with 145,520 products from 64 vendors and search is working great. Here's what's been done since the last update and what's still coming.

Completed — Phase 6: Search Quality (Feb 25, 6:00 PM)

Six improvements to how search understands queries and learns from feedback.

  • Category detection activated — When you search for "brass pendant light," the system now detects it as "lighting" and boosts results from your preferred lighting vendors. This was built but disconnected — now fully wired.
  • Smarter query handling — Specific queries get one focused search. Vague queries like "statement pieces for a living room" get split into 2–3 sub-searches by product type (accent chair, lamp, decorative object).
  • Query-aware feedback — Thumbs up/down votes are now tied to what you searched for. A thumbs-down on a "console table" search penalizes that product most on similar future searches, not unrelated ones.
  • Conversation memory — Products you thumbs-down are automatically excluded from future results in that same conversation.
  • Comment storage fixed — "Why was this a poor result?" text now properly saved (was silently dropped before).
  • Reranker sees full intent — The Cohere reranker now receives your original message instead of the AI's distilled keywords. For "console table with space for ottomans," it understands you want tables, not ottomans.

Feedback System

New in v2

A learning loop that improves search results over time based on user feedback.

How It Works

Hover over any product card to reveal feedback buttons. Thumbs up is instant. Thumbs down opens a comment box asking "Why was this a poor result?" Feedback is fire-and-forget — no loading spinners, no interruption to your workflow.

1
Search
User enters a natural language query
2
Rate
Thumbs up/down on each result card
3
Learn
Vote stored with query context & embedding
4
Improve
Similar searches boosted/penalized, rejected items excluded

New in Phase 6: Feedback is now query-aware. A thumbs-down on a "console table" search penalizes that product mainly for similar furniture searches, not for unrelated "throw pillow" searches. Products you reject are also hidden for the rest of that conversation.

  • Vendor tier system — Top 5 vendors boosted 1.3x, Top 10 boosted 1.2x, Top 20 boosted 1.1x
  • Category priority system — per-category vendor boost/exclusion across 12 product categories
  • Feedback loop — thumbs up/down with free-text comments, query-contextualized scoring, conversation-scoped exclusion
  • Vendor diversity filter — max 3 products per vendor to ensure variety in results
