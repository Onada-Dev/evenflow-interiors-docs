The Big Picture

Your product search engine is live with 145,749 products from 64 vendors. Every single product has been AI-enriched, properly categorized, and embedded for search. Here's what's been done since the last update.

Completed — Phase 11: Vendor Data Fixes (Mar 3)

Three vendor-specific data issues identified and fixed.

  • Surya URLs fixed — All 18,458 Surya product links were returning 404 errors because they were missing a /Product/ segment in the URL path. All 13,152 affected URLs have been corrected and now link directly to the correct product page on Surya's website.
  • Surya names improved — 1,766 Surya products that previously showed inventory codes (like "DNH-002") now have proper names with collection and product type (e.g., "Argil Furniture (AGI-004)"). 16,692 still have inventory codes — fixing the rest requires rendering Surya's JavaScript-heavy pages via Firecrawl (~18K credits, planned as a future improvement).
  • Savoy House images restored — All 1,973 Savoy House products were showing "IMAGE COMING SOON" placeholders even though the real photos exist on Savoy House's server. The issue was expired cache tokens in the image URLs. All product photos have been restored.

Completed — Phase 10: Product Data Normalization (Mar 2)

AI extracted structured attributes for every product — subcategory, colors, materials, styles, and features. This enables precise filtering that wasn't possible before.

  • 99.9% subcategory coverage — Every product is classified into a specific type like "coffee_cocktail_tables," "chandeliers," "sofas_sectionals," etc. (~40 subcategories across 13 categories). Previously, the system matched by name keywords and missed ~42% of products with creative names. Now a search for "cocktail tables" catches ALL cocktail tables, not just ones with "cocktail" in the name.
  • Color filtering works — 83.8% of products tagged with standardized colors (Neutral, Black, White, Grey, Brown, Blue, Green, Red, Metallic, etc.). You can now search "blue velvet sofas" or "brass pendant lights" and get color-accurate results. Products are filtered by color structurally, not by keyword guessing.
  • Material families — 91.0% tagged with normalized materials (Wood, Metal, Glass, Stone, Fabric, Leather, etc.). "Oak" and "walnut" both map to "Wood," so "wood dining tables" catches everything.
  • Clean style tags — 97.4% tagged with curated styles: Modern, Transitional, Traditional, Mid-Century, Coastal, Industrial, etc. The old style_tags field was polluted with junk like "Not In Stock" and vendor codes. These new tags are validated against an approved list.
  • Feature detection — 63.6% tagged with functional attributes: Swivel, Dimmable, Performance Fabric, Modular, Storage, Outdoor-Rated, Handmade, etc. Only extracted when explicitly stated in the product description.
  • Done via Anthropic Batch API (Claude Haiku) for ~$5 total. Each product's existing AI-enriched description was read and classified into structured fields.

Completed — Phase 9: Category Intelligence & Embedding Fix (Feb 28)

A major fix that made 83% of the catalog visible to category-filtered searches, plus improved dimension-based search.

  • 99.5% of products now properly categorized — Previously, each vendor used their own naming scheme for categories (e.g., "mirrors," "wall mirrors," "mirror," "floor_mirror," "Mirrors" — all different). The search engine required an exact match on 12 standard category names, so 83% of products were invisible when you filtered by category. We built a mapping system that translates all 8,260 vendor-specific category names into the 12 standard categories. Now 145,002 of 145,749 products are properly classified.
  • Category filters actually work now — Before this fix, filtering by "Lighting" only showed ~2,000 products. Now it correctly shows all ~18,000 lighting products. Same for every other category. This is the single biggest improvement to search coverage since launch.
  • Dimension search fixed — The AI-enrichment process was accidentally dropping product dimensions from the search index. A "48 by 36 mirror" search couldn't match on size because the system didn't "see" those numbers. Now dimensions are always included, and searches like "mirrors 48 by 36 inches" return 4,099 results instead of 4.
  • Products without dimensions still appear — If you search "console table under 32 inches," products that don't have dimension data won't be excluded — they'll just rank lower than products that match the size filter exactly. This way you still see the full breadth of options.

Why This Isn't Just a Regular Searchable Database

A regular product database (like filtering on Wayfair or Amazon) can only match exact words. If you search "coastal bedroom furniture," it will only find products that literally contain those words in their name or description. Most vendor catalogs don't describe their products that way — a bed might be called "Harbour Cane Poster Bed" with no mention of "coastal" anywhere.

LuxeScout works differently. AI has read every single product listing (all 145,749) and written a search-optimized description that captures what the product is, what style it fits, where it belongs in a home, and what makes it special. That "Harbour Cane Poster Bed" now has context like "coastal-inspired bedroom furniture with natural rattan cane detailing, suited for relaxed beach house or tropical interiors."

On top of that, every product is converted into a mathematical "fingerprint" (an embedding) that captures its meaning, not just its words. So searching "statement pieces for a living room" finds dramatic chandeliers, bold accent chairs, and oversized art — even though none of those products say "statement piece" in their listing.

The result is that you can search the way you naturally think about design — by style, mood, function, or specific dimensions — and get relevant results from across all 64 vendors at once. A traditional database simply can't do that.

Completed — Phase 8: Contextual Enrichment (Feb 27)

AI read every single product listing and wrote a search-optimized summary for each one.

  • 100% AI-enriched — All 145,749 products now have AI-generated contextual descriptions that capture product type, style, room suitability, and distinctive features. This was a one-time batch processing job using the Anthropic Batch API.
  • Vague searches work much better — Queries like "statement pieces for a living room" or "coastal bedroom furniture" now return relevant results because the system understands style and context, not just keywords.
  • Every product benefits — Even products with minimal vendor descriptions now have rich, searchable context. The AI fills in gaps that vendors left out.

Completed — Phase 7: Dimension Search & Result Accuracy (Feb 26)

Two improvements: you can now search by product size, and unrelated products no longer sneak into results.

  • Search by dimensions now works — Previously, searching "mirrors 48 by 36 inches" only returned 4 results because the search engine couldn't see product dimensions. Now it can — over 50,000 products have searchable size data. You can search by width, height, or depth, and product cards now show dimensions right on them.
  • Size filters — The AI now understands size requests like "under 32 inches tall" or "at least 60 inches wide" and automatically filters results to match those constraints.
  • Dimensions on product cards — When a product has size data, you'll see it displayed on the card (e.g., "40.0H x 30.0W x 2.5D") so you don't have to click through to the vendor site just to check measurements.
  • Fixed: unrelated products in results — Searching "cocktail tables" was returning some trays and accessories mixed in, because the word "cocktail" appeared in their descriptions (e.g., "perfect for serving cocktails"). The system now applies product-type filtering to both search methods, so a search for "cocktail tables" returns tables, not trays.

Completed — Phase 6: Search Quality (Feb 25, 6:00 PM)

Six improvements to how search understands queries and learns from feedback.

  • Category detection activated — When you search for "brass pendant light," the system now detects it as "lighting" and boosts results from your preferred lighting vendors. This was built but disconnected — now fully wired.
  • Smarter query handling — Specific queries get one focused search. Vague queries like "statement pieces for a living room" get split into 2–3 sub-searches by product type (accent chair, lamp, decorative object).
  • Query-aware feedback — Thumbs up/down votes are now tied to what you searched for. A thumbs-down on a "console table" search penalizes that product most on similar future searches, not unrelated ones.
  • Conversation memory — Products you thumbs-down are automatically excluded from future results in that same conversation.
  • Comment storage fixed — "Why was this a poor result?" text now properly saved (was silently dropped before).
  • Reranker sees full intent — The Cohere reranker now receives your original message instead of the AI's distilled keywords. For "console table with space for ottomans," it understands you want tables, not ottomans.

Feedback System

New in v2

A learning loop that improves search results over time based on user feedback.

How It Works

Hover over any product card to reveal feedback buttons. Thumbs up is instant. Thumbs down opens a comment box asking "Why was this a poor result?" Feedback is fire-and-forget — no loading spinners, no interruption to your workflow.

New in Phase 6: Feedback is now query-aware. A thumbs-down on a "console table" search penalizes that product mainly for similar furniture searches, not for unrelated "throw pillow" searches. Products you reject are also hidden for the rest of that conversation.
